    介绍⚓︎

    SQLite 是一个提供关系型数据库管理系统的软件库。名称中的 "lite" 表现的是 SQLite 在设置、数据库管理和需求资源等方面的轻量化 (lightweighted) 本质。具体来说，SQLite 具备以下显著特点：

    • 自给 (self-contained)：不需要操作系统或外部库提供很多资源，这使得它能够在任何环境下（包括嵌入式系统）运行，可移植性高。
    • 无需服务器 (severless)：一般的 RDBMS，比如 MySQL 和 PostgreSQL 需要一台单独的服务器，应用程序通过 TCP/IP 协议的发送和请求来访问数据库服务器。然而，SQLite 的数据库就集成在应用程序中，应用程序可以直接从数据库中读写数据。
    • 零配置 (zero-configuration)：正因为 SQLite 的无服务器架构，因此我们无须事先配置 SQLite
    • 事务处理的 (transactional)：SQLite 的事务遵守 ACID，即具备原子性 (atomic)、一致性 (consistent)、隔离性 (isolated) 和持久性 (durable) 的特征

    此外，SQLite 还具备以下独特之处：

    • 使用动态类型，因此能在表中的任何列存储任何值，可无视声明的类型
    • 允许通过单个数据库连接来同时访问多个数据库文件
    • 由于 SQLite 创建的数据库位于内存中，因此原型设计 (prototyping) 和测试的速度会非常快

    何时使用 SQLite 呢？

    • 嵌入式应用
    • 本地存储：SQLite 很适合那些需要在本地存储用户偏好、设置和缓存的应用
    • 跨平台应用：SQLite 支持多平台运行
    • 快速的原型设计和开发
    • 物联网 (IoT) 设备：由于 IoT 使用资源有限，因此轻量级的 SQLite 能胜任这种情况

    下载 & 安装⚓︎

    前往官方的下载页，根据自己的电脑类型下载合适的版本。下面以 Windows 版本和 Linux 版本为例：

    1. 在下载页中找到 Windows 版本，点击下图红色方框处的链接下载。

    2. 下载好后解压压缩包，得到文件夹，然后点击进入。

    3. 按住 Shift 键，然后右键点击文件夹里的空白部分，呼出 Windows 10 风格的菜单，点击带有“终端”字样的选项。

      熟悉命令行操作的小伙伴可以用 cd 命令切换到 sqlite 所在目录，无需遵守 2, 3 两步

    4. 在终端命令行中输入 sqlite，如果输出如下所示的信息，则说明 SQLite 可以正常运行。

    5.（可选，但十分推荐）退出 SQLite 程序（输入 .quit 命令），将 sqlite 所在目录迁移到合适的位置，然后设置环境变量，使得 SQLite 在电脑的任何目录下都可以运行。

    • 按下 Win + R 键，打开“运行”界面，输入 sysdm.cpl 后敲回车，进入“系统属性”界面

    • 点击该界面的“高级”选项卡，然后点击“环境变量”按钮

    • 在“环境变量”界面选中 Path 并双击，编辑该环境变量。然后在该界面新建项，内容为 SQLite 程序的目录（我这里放在 D:\sqlite 目录下）

    • 最后不要忘记点击三个“确定”按钮，依次退出上述打开的界面，这样才能保存我们的设置

    • 重新打开终端，输入 sqlite3，如果不出意外的话，在任何目录下应当都可以启动 SQLite 了（现在我在 HOME 目录下也能打开 SQLite 了）！

    超级简单，只要输入以下命令即可：

    $ sudo apt update               # 更新
$ sudo apt install sqlite3      # 不要忘记最后的 3
$ sqlite3                       # SQLite，启动！

    如果出现类似下面的打印信息，说明安装成功啦！

    SQLite version 3.45.3 2024-04-15 13:34:05
Enter ".help" for usage hints.
Connected to a transient in-memory database.
Use ".open FILENAME" to reopen on a persistent database.
sqlite>

    此外，SQLite 还有 GUI 工具，比如：

    • SQLiteStudio
    • DBeaver
    • DB Browser for SQLite

    不过，鉴于笔者时间有限，外加笔者喜欢用命令行访问数据库的习惯（感觉这样更像一名 CSer），所以这些工具的安装和使用就不再介绍了，对此感兴趣的读者可以点击这些链接或者在网上搜索相关教程进行学习。

    命令行环境⚓︎

    最开始，我们需要在命令行环境执行 sqlite3 path/to/my_database.db 指令，用 SQLite 打开已经存在的数据库文件，或者创建不存在的数据库文件。如果命令行的开头变成了 sqlite>，说明 SQLite 成功启动。

    在 SQLite 中，所有的命令都是以 . 开头的。这里先介绍一些最基本，也是最常用的 SQLite 命令：

    • .open .open FILENAME：也可以在 SQLite 内打开数据库文件

    • .databases：显示当前 SQLite 连接的数据库，它至少会显示一个名为 main 的数据库

      • 如果一个数据库都没有，会显示：

        sqlite> .databases
main: "" r/w

      • 可以使用 attach 语句将本地数据库文件连接到 SQLite 上：

        attach database "c:\sqlite\db\chinook.db" AS chinook;

      • 现在就有一些新的数据库了：

        sqlite> .databases
main: "" r/w
chinook: c:\sqlite\db\chinook.db r/w

    • .tables：显示数据库中所有的表

      • 该命令支持模式 (pattern) 匹配（类似 GNU Make），比如 % 表示一个或多个字符，那么 %es 就表示以 es 结尾的字符串，因此：

        sqlite> .open C:\Users\qzy\Downloads\chinook\chinook.db
sqlite> .tables %es
employees    genres       invoices     media_types

    • .schema TABLE：显示名为 TABLE 的表的结构，即模式 (schema)

      例子 
      sqlite> .schema albums
CREATE TABLE IF NOT EXISTS "albums"
(
    [AlbumId] INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
    [Title] NVARCHAR(160)  NOT NULL,
    [ArtistId] INTEGER  NOT NULL,
    FOREIGN KEY ([ArtistId]) REFERENCES "artists" ([ArtistId])
                ON DELETE NO ACTION ON UPDATE NO ACTION
);
CREATE INDEX [IFK_AlbumArtistId] ON "albums" ([ArtistId]);
      • 如果只输入 .schema，那么就会显示所有表的结构
      • 如果想要查看所有表，包括与 SQLite 状态相关的表的结构及其内容，可以输入 .fullschema 命令

    • indexes：显示当前数据库的全部索引

      • indexes TABLE：只显示名为 TABLE 的表的索引
    • .help：打印帮助信息，显示各种 SQLite 命令以及选项
    • .quit：退出 SQLite 命令行界面

    • 格式化查询输出相关的命令：

      • .mode column：将输出模式设置为 column
      • .header on：显示列名
      • .nullvalue NULL：设置空值字面量

      这样可以使输出的内容更加美观。

    • .output FILENAME：执行该命令后，之后的查询结果都会保存在 FILENAME 文件下（可指定路径）

      • .output 命令会重新让查询结果显示在标准输出（即命令行）中

    • .read FILENAME：执行保存在 FILENAME 内的 SQL 语句

      • 对于多行、重复的 SQL 语句而言，这个功能就很有用了

    在 SQLite 中，我们可以输入任何合法的 SQL 语句。

    • SQL 语句可以拆成多行，敲回车可以切换到下一行，用分号表示一条语句的结束，比如：

      sqlite> select *
    ...> from albums
    ...> order by AlbumId
    ...> limit 3;

      再敲回车的话，SQLite 就开始执行这条语句，并给出相应的输出。

    • 如果敲到一半，发现语句写错的话，那么很遗憾，SQLite 不支持退格到上面的行，所以若发现前面行的错误的话，就不得不提前结束，从头再来——输入分号敲回车，提前结束 SQL 语句的编写。

    基础语法⚓︎

    Queries⚓︎

    SELECT 语句是 SQL 中最常用的语句之一，而 SQLite 的 SELECT 语句提供了 SQL 标准下的全部功能。

    • 支持简单的计算：

      sqlite> SELECT
    ...> 10 / 5,
    ...> 2 * 4;
2|8

    • 查询数据：下面给出 SELECT 语句的基本语法：

      SELECT DISTINCT column_list
FROM table_list
JOIN table ON join_condition
WHERE row_filter
ORDER BY column
LIMIT count OFFSET offset
GROUP BY column
HAVING group_filter;
      • DISTINCT 子句 (clause) 用于查询表中唯一的一行记录
      • JOIN 用于从多张表中进行查询
      • WHERE 子句用于在结果集合中过滤行记录
      • ORDER BY 子句用于对结果集合排序
      • LIMIT 偏移量子句用于限制返回的行数
      • GROUP BY 子句用于对行分组，以便使用聚合函数 (aggregate functions) 计算
      • HAVING 子句用于过滤组别

    这里只是简单罗列了一下 SELECT 语句包含了哪些可选的 (optional) 子句，无需深究它们的功能，在后面的章节中会一一介绍的。现在让我们先仅专注于 SELECT 语句本身，即：

    SELECT column_list
FROM table;
    • 注意细节问题：SQLite 会先求解 (evaluate) FROM子句，然后再求解 SELECT 子句
    • 在 column_list 中，我们可以指明一个或多个列（用逗号间隔）；在 table 中，我们可以指明一张或多张表（用逗号间隔）
    • 如果 column_list 处用 * 表示，那么表示选择表格的所有列
      • 注意：在实际编写程序时，不要采用这种方式，因为如果其他开发者增删了某些列，那么使用这种方法得到的结果就和原来的不一样了，因而产生 bug。所以，请还是老老实实地选择需要用到的列！

    Sorting⚓︎

    SQLite 在存储行记录的时候不会指定顺序，这就意味着表格的行记录可能是按照插入顺序排序的，也可能不是，所以直接使用 SELECT 语句得到的行记录也是没有指定顺序的。如果想要得到有顺序的查询结果，需要在 SELECT 语句中使用 ORDER BY 子句，比如：

    SELECT
   select_list
FROM
   table
ORDER BY
    column_1 ASC,
    column_2 DESC;
    • ORDER BY 子句紧随 FROM 子句后面，可以基于单列或多列（还是以逗号间隔），按照升序（ASC）或降序（DESC）的顺序来对结果集合排序
    • 如果没有指定 ASC 或 DESC 关键字，SQLite 会默认采用升序排序（ASC）
    • ORDER BY 子句会按照指定列的先后顺序进行排序。对于上例，SQLite 先基于 column_1 升序排序，然后基于 column_2 降序排序
    • ORDER BY 子句中的列不一定要出现在 select_list 中，只要是同一张表的列都没问题
    • 在 ORDER BY 子句中，除了可以指定具体的列名外，还可以用列的位置表示列。比如 column_1 是第 3 列，那我们就可以用 3 代替 column_1
    • 在排序中，特殊值 NULL 小于任何值，因此在升序排序中，NULL 会出现在结果集合的开头（即在其他类型的值之上）；而在降序排序中，NULL 会出现在结果集合的末尾（即在其他类型的值之下）
      • 在 SQLite 3.30.0 版本中，引入关键字 NULL FIRST 和 NULL LAST，顾名思义，就是分别指定 NULL 值时钟出现在结果集合的开头和末尾，无视原来的排序规则

    Filtering⚓︎

    下面介绍一些与过滤条件相关的 SQLite 子句或关键字：

    • DISTINCT 子句：去除结果集合中的重复行记录，仅保留其中一项
      • 它紧跟在 SELECT 关键字之后
      • 虽然在数据库中 NULL != NULL，但是 DISTINCT 会将其视为重复数据，因此仅保留一行带 NULL 值的记录

    • WHERE 子句：指定查询条件，语法为：

      SELECT
    column_list
FROM
    TABLE
WHERE
    search_condition;
      • SQLite 的求解顺序为：FROM -> WHERE -> SELECT

      • 具体来说，查询条件可以是用逻辑运算符连接的一个或多个比较表达式

        例子 
        WHERE column_1 = 100;
WHERE column_2 IN (1,2,3);
WHERE column_3 LIKE 'An%';
WHERE column_4 BETWEEN 10 AND 20;

      • WHERE 子句还可以用在 UPDATE、DELETE 等语句中

      • SQLite 的比较运算符有：=（等于）、<>/!=（不等于）、<、>、<=、>=

      • SQLite 的逻辑运算符有：

        • ALL：若所有的表达式（3 个以上）为 true，则返回 true
        • AND：若所有的表达式（2 个）为 true，则返回 true；只要有一个为 false 就返回 false
          • 如果一个表达式为 true，另一个表达式为 NULL；或者两个表达式都为 NULL 时，结果为 NULL
        • ANY：只要有一个表达式为 true，则返回 true

        • BETWEEN：如果值在指定范围内，则返回 true，具体语法为：

          test_expression BETWEEN low_expression AND high_expression

          等价于以下表达式：

          test_expression >= low_expression AND test_expression <= high_expression
          • 一般情况下，low_expression 的值 <= high_expression，否则的话 BETWEEN 永远返回 false
          • 如果表达式中出现 NULL，结果就是 NULL

          • 可以使用 NOT 取反，此时表达式等价于：

            test_expression < low_expression OR test_expression > high_expression

        • EXISTS：若一个子查询包含任意行记录时，返回 true

        • IN：若值出现在指定列表或子查询内，则返回 true，具体语法为：

          expression [NOT] IN (value_list|subquery);

          其中 value_list 是用逗号间隔的一系列定值，而 subquery 则是与表达式类型相同的单列子查询结果

        • LIKE 子句：如果值与指定模式匹配，则返回 true，从而实现模糊匹配，具体语法为：

          SELECT
    column_list
FROM
    table_name
WHERE
    column_1 LIKE pattern;

          • SQLite 提供了两类通配符 (wildcards)，经常会在 LIKE 子句中使用：

            • %：匹配 0 个或多个字符。比如 %per% 表示包含子字符串 per 的所有字符串
            • _：匹配任意单个字符。比如 h_nt 能匹配 hunt、hint 等

          • LIKE 对大小写不敏感，可以使用以下语句调整大小写敏感的选项

            PRAGMA case_sensitive_like = true;

          • 如果希望模式能够匹配包含 % 或 _ 这些字面量，需要配合 ESCAPE 子句构造模式，具体语法为：

            column_1 LIKE pattern ESCAPE expression;

            此时，pattern 中紧跟在 expression 求解值后面的字符会被视为一般字面量，而不会被看做通配符

            例子 
            column_1 LIKE '%10\%%' ESCAPE '\';

            能够匹配包含 10% 的子字符串。

        • NOT：与其他运算符配合，逆转其值，比如 NOT EXISTS、NOT IN、NOT BETWEEN 等等

        • OR：只要有一个表达式为 true，则返回 true；若所有表达式（2 个）为 false，则返回 false

          • 如果一个表达式为 false，另一个表达式为 NULL；或者两个表达式都为 NULL 时，结果为 NULL

        • GLOB：功能与 LIKE 很像，但是有以下区别：

          • 对大小写敏感
          • 使用 UNIX 通配符：
            • *、? 分别对应 LIKE 的 % 或 _
            • []：用于匹配在列表中的一系列字符，比如 [a-zA-Z0-9] 可以匹配所有的单个字母和数字
            • ^：用于列表开头，表示排除指定字符，比如 [^0-9] 可以匹配除数字字符外的所有字符
          • 不使用 ESCAPE 从句

        • IS NULL：检验列或表达式是否为 NULL

          • 之所以出现这种运算，是因为 NULL 不等于任何值，包括它自身，所以 = NULL 的结果永远为 false。但我们希望找到值为 NULL 的结果，于是有了这种运算。
          • 也可以结合 NOT，即 IS NOT NULL，寻找值不为 NULL 的记录

    • LIMIT 子句：限制结果集合的行数，具体语法为：

      SELECT
    column_list
FROM
    table
LIMIT row_count;

      其中 row_count 是一个正整数，表示返回行数。如果想要从某一行开始返回多行，则语法为：

      SELECT
    column_list
FROM
    table
LIMIT row_count OFFSET offset;

      也可以简写为：

      SELECT
    column_list
FROM
    table
LIMIT offset, row_count;

      其中 offset 表示偏移量，此时返回结果就会从 offset + 1 行开始计数。
